Recharge ​

ClawdSaber uses a credit system to pay for model usage. When your credits run low, you can top up from the Recharge page.

What the Recharge Page Offers ​

  • Current credit balance display
  • Amount selection
  • Order creation and payment flow
  • Order history
  • Model pricing reference

Step 1: Select an Amount ​

When you open the Recharge page, you will see preset amounts:

  • 10 USDT
  • 50 USDT
  • 100 USDT

You can also enter a custom amount.

Limits ​

  • The minimum recharge amount is 1 USDT.
  • The page estimates how many credits you will receive based on the current exchange rate.

Step 2: Create an Order ​

After selecting an amount, click the recharge button to create an order.

The app switches to the payment waiting screen, which shows:

  • Receiving address
  • Amount to send
  • Network
  • Estimated credits
  • Order number
  • Remaining time

Step 3: Send Payment ​

Send the exact amount to the displayed address using the correct network.

The two most important things:

  1. Amount must match exactly.
  2. Network must be correct.

Using the wrong network may result in lost funds, even if the address looks correct.

Step 4: Wait for Confirmation ​

The Recharge page automatically polls the order status. You do not need to refresh manually.

Additional behaviors:

  • If you return to the Recharge page while a pending order exists, the app resumes that order automatically.
  • After a successful payment, your credit balance updates automatically.

Order History ​

The bottom of the Recharge page shows your order history.

Orders have three possible statuses:

  • Credited — payment received and credits added
  • Expired — the order timed out before payment was detected
  • Pending — waiting for payment

You can click a pending order to return to its payment screen.

Model Pricing Reference ​

The Recharge page also includes a model pricing table that shows:

  • Official price
  • Platform price
  • Discount percentage
  • Input credit cost
  • Output credit cost

This is useful for estimating how far your credits will go.

Common Mistakes ​

Amount too small ​

Orders cannot be created for amounts below 1 USDT.

Wrong network ​

Always use the network displayed on the payment screen. Do not mix networks.

Order expired ​

If an order expires, you must create a new one. Expired orders cannot be reused.

If you encounter payment issues, check the FAQ.
